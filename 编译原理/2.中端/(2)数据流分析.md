数据流分析（DataFlow Analysis）
===============================

数据流分析是一种重要的代码分析思路，它不是一种具体的分析手段，而是一类分析方法。通过数据流分析，可以分析程序中的数据流，找出程序中的数据流特性，比如变量的定义和使用关系、变量的活跃性、变量的赋值关系等。它可以用于很多不同的分析任务，比如指针分析、内存泄漏检测、内存访问越界检测等。


## 数据流分析的基本思想

如果把一段程序画成控制流图，那数据流分析干的事，本质上就是**沿着图的边，把某种信息一路传下去**。

你可以把它想成编译器里的「八卦传播系统」：

- 每个基本块门口站着一个保安，负责汇报：「我这里定义了谁」「谁还活着」「哪个表达式已经算过了」
- 顺着边往下走，信息就像小道消息一样被带过去
- 遇到 `if/else` 汇合的地方，两边带来的消息得合并一下——就像两个同学分别听说「小明在教室」和「小明在操场」，汇合点得整理成「小明可能在教室或操场」
- 如果某条路还能绕回来（循环），消息可能越传越全，于是大家**反复传，直到没人有新料可爆**——这就是不动点迭代

所以数据流分析并不神秘：先定好「要传播什么信息」，再定好「进门怎么更新、出门怎么合并」，最后让信息在 CFG 上跑几圈，直到稳定下来。活跃变量、到达定义、可用表达式，其实都是同一套套路，只是传的「八卦内容」不一样罢了。

## 数据流分析的应用

### 可达表达式分析（Available Expression Analysis）

鲁迅曾说：「如果我们已经算过了，那我们为什么要再来一遍？」

所以，我们可以通过数据流分析，来找出程序中哪些表达式已经计算过了，从而避免重复计算。



### 活跃变量分析（Live Variable Analysis）

薛定谔的变量 - 在你进行活跃变量分析前，一个变量处于「既死又活」的叠加态。分析完之后，它要么活着，要么死了，坍缩了。

一个变量什么时候能被称为「活着」？

- 在它被赋值之前，它不能被使用
- 在它被赋值之后，它可以在它作用域内的任何地方被使用
- 在它作用域结束之后，它不能被使用





### 到达定义分析（Reaching Definition Analysis）

到达定义分析想回答一个「追根溯源」的问题：**程序执行到某一点时，某个变量当前的值可能是被哪几条赋值语句写进去的？**

这里的「定义」指的是一次赋值（比如 `x = 5` 就是对 `x` 的一个定义）。如果从这条赋值出发，存在一条路径能走到某个程序点，且中途 `x` 没有被重新赋值「盖掉」，我们就说这个定义**到达（reach）** 了那个点。

沿用前面「八卦传播」的比喻：

- 每条赋值语句 `x = ...` 都会**产生（gen）** 一条新八卦：「x 现在归我管」
- 一旦后面又出现 `x = ...`，就会**杀死（kill）** 之前所有关于 x 的旧八卦——毕竟 x 被重新写过了
- 顺着 CFG 往下传，汇合点用**并集**合并：只要有任意一条路径能带来某个定义，它就算能到达

用公式表达，每个基本块的传播规则是：

```
OUT[B] = gen[B] ∪ (IN[B] − kill[B])
IN[B]  = ∪ OUT[P]   （P 是 B 的所有前驱）
```

它是典型的**前向 + 并集**数据流分析。分析结果很实用：如果某个使用点只被唯一一个定义到达，就能做常量传播；如果一个定义谁都没到达（没被用过），那它多半是死代码。

### 指向分析（Point-to Analysis）

指向分析想回答一个「指针指向哪」的问题：**程序执行到某一点时，某个指针可能指向哪些对象？**

前面几种分析传播的是「变量活不活」「定义到没到」，而指向分析传播的八卦升级了：不再是简单的「有/无」，而是一张 **「谁指向谁」的花名册**。

- 遇到 `p = &a`，就记一笔：`p → {a}`（p 可能指向 a）
- 遇到 `p = q`，就把 q 的花名册抄给 p：`q → {a}` 时 `p` 也得到 `{a}`
- 遇到 `*p = ...` 或 `x = *p`，还得顺着 p 的花名册去更新它指向的那些对象

由于程序有分支和循环，一个指针往往**可能指向多个对象**，所以结果通常是一个集合（可能指向 a 或 b），而不是一个确定答案。这也是它比前几种分析更「烧脑」的地方。

分析的精度取决于取舍：是否区分调用上下文（上下文敏感）、是否区分控制流顺序（流敏感），越精细越准，但也越慢。具体的算法（基于图、基于约束等）可以看 [指向分析](./指向分析.md) 一章。它的结果是别名分析、内存泄漏检测等一系列分析的基石。



### 死代码消除（Dead Code Elimination）


死代码消除想回答一个「哪些代码没用」的问题：**程序执行到某一点时，哪些代码不会被执行到？**
